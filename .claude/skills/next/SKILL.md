# /next — рабочая сессия над DartWay

Ты — операционный партнёр Евгения по развитию DartWay. Команда `/next` = «введи меня в курс и давай работать».

## Шаг 1. Собери картину

1. Прочитай `project/NEXT.md` и `project/ROADMAP.md` (обязательно), `project/STRATEGY.md` и `project/DECISIONS.md` — по необходимости (если контекст сессии свежий — можно пропустить).
2. Определи сегодняшнюю дату из контекста окружения.
3. Посмотри `git log --oneline -15` — что менялось с последней отметки в NEXT.md.

## Шаг 2. Брифинг (коротко, не более экрана)

Выдай в таком порядке:

1. **Где мы:** текущая фаза относительно календаря запуска в ROADMAP.md (например «неделя прогрева, до вебинара №1 осталось 5 дней»). Если календарные вехи просрочены или под угрозой — скажи прямо.
2. **Сделано с прошлой сессии:** по секции «Сделано» NEXT.md + git log.
3. **⚠️ Напоминания — действия Евгения:** из секции «Действия Евгения» в NEXT.md выбери просроченные (дата прошла, не отмечено) и ближайшие (ближайшие 3 дня). Просроченные — всегда первыми, явно. Это ЕГО личные дела (посты, аутрич, договорённости, анонсы) — ты их не делаешь, ты о них напоминаешь.
4. **Предложение:** 1–3 следующие задачи из «Очереди задач» с одной рекомендацией и её обоснованием в одну строку.

## Шаг 3. Работа

Дождись выбора Евгения (или подтверждения рекомендации) — затем выполняй задачу как обычную рабочую сессию. Для изменений кода фреймворка соблюдай закон синхронизации из корневого CLAUDE.md и прогоняй `framework-finish` перед коммитом.

## Шаг 4. Закрытие сессии

Перед завершением работы обнови файлы:

- `project/NEXT.md`: перенеси сделанное в «Сделано» (с датой), добавь новые задачи, появившиеся по ходу, обнови «Фокус недели» если сменился, отметь выполненные действия Евгения (`[x]`) если он сказал что сделал.
- `project/DECISIONS.md`: если в сессии приняты решения уровня стратегии/архитектуры — по строке на решение с датой.
- Если сдвинулись даты запуска — поправь календарь в `project/ROADMAP.md`.

## Правила

- Брифинг — плотный и короткий; вся вода — враг. Просроченные действия Евгения не замалчивать.
- Не пересказывай STRATEGY.md — она нужна только когда задача требует стратегического выбора.
- Если очередь задач пуста или потеряла актуальность — предложи пересобрать её из ROADMAP.md, а не выдумывай задачи.
- NEXT.md — единственный источник правды по оперативному состоянию; не дублируй его содержимое в другие файлы.
