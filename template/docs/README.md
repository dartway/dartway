# Documentation

- **`1_general/`** — architecture and infrastructure of this project.
- **`2_features/`** — one doc per feature, kept in sync with its code. Copy
  `_TEMPLATE.md` when you add a feature.
- **`audits/`** — reports produced by `/dartway-audit`.

The DartWay methodology itself lives in `.claude/` (CLAUDE.md + skills). These
docs hold what is specific to *this* project. Law of the toolkit: change a
feature's code, update its doc in the same task (`dartway-finish` checks this).
